# 副业评估决策工具 (Side Hustle Evaluator)

## 概述

本技能提供一套**理性的副业评估体系**，帮助用户系统性地判断一个副业是否值得投入。通过四个步骤——自我评估、项目分析（SWOT）、价值核算、风险预判——从内到外、从定性到定量，做出明智的决策。

**核心原则：不凭感觉，不盲目跟风，用数据和框架说话。**

---

```
用户提出副业方向
       │
       ▼
┌──────────────────────────────────┐

│ scripts/profile.py               │
└────────┬─────────────────────────┘
         │ 画像文件已存在？
         ├── 是 → 加载画像，询问是否需要更新
         └── 否 → 引导收集信息，创建画像文件
         │
         ▼
┌──────────────────┐
│ 第二步：SWOT分析  │ ← 基于画像 + 副业方向
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 第三步：价值核算  │ ← 画像中的主业数据自动填入
└────────┬─────────┘
```

**⚠️ MANDATORY：第一步产出的用户画像会保存为 `user-profile.json` 文件。后续每次评估副业时，直接加载已有画像，无需用户重复输入。仅在用户主动要求更新或画像不完整时才重新收集信息。**

## 第一步：自我评估 — 生成用户画像文件

**目标：** 一次性收集用户的基本情况，生成持久化的画像文件。后续评估直接复用，避免重复输入。

**核心机制：** 画像文件 (`user-profile.json`) 是一次创建、多次复用、可增量更新的持久化文件。

### 1.0 画像文件管理（脚本驱动）

**⚠️ MANDATORY：** 第一步必须使用 `scripts/profile.py` 脚本管理画像文件，支持以下操作：

| 操作 | 命令 | 场景 |
|------|------|------|
| **创建** | `python3 scripts/profile.py create --json '{...}'` | 首次使用，收集信息后创建 |
| **加载** | `python3 scripts/profile.py load` | 已有画像，直接加载复用 |
| **更新** | `python3 scripts/profile.py update --json '{...}'` | 用户信息有变化，增量更新 |
| **校验** | `python3 scripts/profile.py validate` | 检查画像完整性和有效性 |

### 1.1 执行流程

每次进入第一步时，按以下流程执行：

```
开始第一步
    │
    ▼
尝试加载已有画像 (profile.py load)
    │
    ├── 画像不存在 ──→ 进入「首次创建」流程 (1.2)
    │
    ├── 画像存在且完整 ──→ 展示画像摘要，询问用户：
    │   "已找到你的画像（上次更新：XX），信息是否有变化？"
    │   ├── 无变化 → 直接进入第二步
    │   └── 有变化 → 进入「更新」流程 (1.3)
    │
    └── 画像存在但不完整 ──→ 提示缺失字段，仅收集缺失部分 (1.3)
```

### 1.2 首次创建画像

通过对话一次性收集以下信息（使用 AskUserQuestion 工具，分批提问）：

**第一批：时间与职业（必填）**

| 字段 | 关键问题 | 选项/格式 |
|------|---------|----------|
| `name` | 你怎么称呼？ | 自由文本 |
| `daily_hours` | 每天能稳定投入多少小时做副业？ | 数值（如 2.0） |
| `time_quality` | 这些时间是碎片化还是整块的？ | 整块 / 碎片化 / 混合 |
| `sustainability` | 你能持续投入多久？ | 短期(1-3月) / 中期(3-6月) / 长期(6月+) |
| `main_job` | 你的主业/职业是什么？ | 自由文本 |
| `main_job_monthly_salary` | 主业月薪资大约多少？ | 数值（元） |
| `main_job_monthly_hours` | 主业每月工作多少小时？ | 数值（如 160） |

**第二批：技能与资源（必填）**

| 字段 | 关键问题 | 选项/格式 |
|------|---------|----------|
| `hard_skills` | 你有哪些硬技能？ | 列表（如 ["Python", "写作"]） |
| `soft_skills` | 你有哪些软技能？ | 列表（如 ["沟通", "管理"]） |
| `interests` | 你对什么领域感兴趣？ | 列表（如 ["科技", "教育"]） |
| `resources` | 你有什么可用资源？ | 列表（如 ["笔记本", "资金5000"]） |

**第三批：家庭与风险偏好（必填）**

| 字段 | 关键问题 | 选项/格式 |
|------|---------|----------|
| `family_support` | 家人是否支持你做副业？ | 已沟通 / 待沟通 / 不支持 |
| `risk_tolerance` | 你的风险偏好？ | 保守 / 稳健 / 激进 |

**对话引导话术：**
> "在评估副业之前，我们先建立你的个人画像。这个画像只需创建一次，以后每次评估副业都可以直接复用。我们先从时间和职业情况开始："

收集完毕后，调用脚本创建画像：

```bash
python3 scripts/profile.py create --json '{
    "name": "[姓名]",
    "daily_hours": [数值],
    "time_quality": "[整块/碎片化/混合]",
    "sustainability": "[短期/中期/长期]",
    "main_job": "[主业]",
    "main_job_monthly_salary": [数值],
    "main_job_monthly_hours": [数值],
    "hard_skills": ["技能1", "技能2"],
    "soft_skills": ["技能1", "技能2"],
    "interests": ["兴趣1", "兴趣2"],
    "resources": ["资源1", "资源2"],
    "family_support": "[已沟通/待沟通/不支持]",
    "risk_tolerance": "[保守/稳健/激进]"
}'
```

### 1.3 更新画像

当用户信息有变化时，仅需收集变化的部分，使用 update 操作增量更新：

```bash
python3 scripts/profile.py update --json '{
    "daily_hours": 3.0,
    "hard_skills": ["Python编程", "数据分析", "视频剪辑"]
}'
```

脚本会返回变更摘要，展示哪些字段被修改了。

### 1.4 画像输出

创建或加载画像后，展示画像摘要（不展示全部字段，只展示关键信息）：

```

👨‍👩‍👧 家庭支持：[已沟通/待沟通/不支持]
```

**CRITICAL：** 画像中的 `main_job_monthly_salary` 和 `main_job_monthly_hours` 会在第三步「价值核算」中自动填入，无需用户再次提供。

## 第二步：项目分析 — SWOT模型看清全貌

**目标：** 对用户选定的副业方向，从四个维度进行全面审视。**SWOT分析中的「优势」和「劣势」应直接引用画像中的技能和资源信息。**

### 2.1 SWOT分析框架

使用 AskUserQuestion 工具逐一收集四个维度的信息，或基于用户已提供的信息进行分析：

| 维度 | 关键问题 | 分析要点 |
|------|---------|---------|
| **优势 (S)** | 我做这个有什么别人没有的优势？ | 技能壁垒、经验积累、资源独特性（**从画像中提取**） |
| **劣势 (W)** | 我的短板是什么？ | 经验不足、不懂引流、资金不足、害怕公开表达 |
| **机会 (O)** | 外部环境有什么红利？ | 市场需求增长、新平台/工具降低门槛、AI赋能 |
| **威胁 (T)** | 可能遇到哪些风险？ | 竞争内卷、平台规则多变、诈骗陷阱、与主业冲突 |

### 2.2 SWOT策略矩阵

基于分析结果，形成四种策略：

| 策略 | 组合 | 说明 | 成功率 |
|------|------|------|--------|
| **SO策略** | 优势 + 机会 | 用自身优势抓住外部机会 | ⭐⭐⭐⭐⭐ 最高 |
| **WO策略** | 劣势 + 机会 | 先弥补劣势再抓住机会 | ⭐⭐⭐ 中等 |
| **ST策略** | 优势 + 威胁 | 用优势抵御外部威胁 | ⭐⭐⭐⭐ 较高 |
| **WT策略** | 劣势 + 威胁 | 最差组合，需慎重考虑 | ⭐⭐ 最低 |

### 2.3 SWOT分析输出

```
📊 SWOT分析报告 — [副业名称]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💪 优势 (Strengths)：
   • [S1]
   • [S2]

📉 劣势 (Weaknesses)：
   • [W1]
   • [W2]

🚀 机会 (Opportunities)：
   • [O1]
   • [O2]

⚠️ 威胁 (Threats)：
   • [T1]
   • [T2]

🎯 推荐策略：[SO/WO/ST/WT]
   策略说明：[具体建议]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 第三步：价值核算 — 算清三笔关键账（脚本精确计算）

**目标：** 这是判断副业"值不值得做"的核心步骤。通过脚本精确计算三笔账。

**⚠️ MANDATORY：必须使用 `scripts/evaluate.py` 脚本进行数值计算，不要手动计算，以保证结果准确。**

### 3.1 收集计算所需数据

通过对话收集以下参数。**其中标注「🔄 从画像自动填入」的字段，直接从 user-profile.json 中读取，无需再次询问用户。**

| 参数 | 说明 | 来源 | 示例 |
|------|------|------|------|
| `side_hustle_name` | 副业名称 | 用户输入 | "自媒体运营" |
| `monthly_gross_income` | 副业月总收入（元） | 用户输入 | 3000 |
| `monthly_hours` | 月投入小时数 | 用户输入 | 60 |
| `main_job_monthly_salary` | 主业月薪资（元） | 🔄 从画像自动填入 | 15000 |
| `main_job_monthly_hours` | 主业月工作小时数 | 🔄 从画像自动填入 | 160 |
| `startup_cost` | 启动成本（元） | 用户输入 | 2000 |
| `monthly_operating_cost` | 月运营成本（元） | 用户输入 | 500 |
| `alternative_hourly_value` | 替代方案时薪（元/小时） | 用户输入 | 100 |
| `alternative_skill_value` | 替代方案的附加价值描述 | 用户输入 | "学习视频剪辑，预计3个月后可接单" |

**对话引导话术：**
> "现在进入最关键的价值核算环节。你的主业信息已从画像中自动读取（月薪 ¥XXXX，月工作 XXX 小时）。我还需要以下副业相关数据，请尽量提供具体数字（估算值也可以）："

### 3.2 执行脚本计算

将收集到的参数组装为 JSON，调用评估脚本：

```bash
python3 scripts/evaluate.py --json '{
    "side_hustle_name": "[名称]",
    "monthly_gross_income": [数值],
    "monthly_hours": [数值],
    "main_job_monthly_salary": [画像中的值],
    "main_job_monthly_hours": [画像中的值],
    "startup_cost": [数值],
    "monthly_operating_cost": [数值],
    "alternative_hourly_value": [数值],
    "alternative_skill_value": "[描述]"
}'
```

**CRITICAL：** 脚本路径为当前 skill 目录下的 `scripts/evaluate.py`。执行时需使用绝对路径或确保工作目录正确。

### 3.3 解读计算结果

脚本返回 JSON 格式的评估结果，包含：

#### 时间账 (time_account)
- `side_hustle_hourly_rate` — 副业时薪（元/小时）
- `main_job_hourly_rate` — 主业时薪（元/小时）
- `hourly_ratio` — 副业时薪/主业时薪比率
- `time_score` — 评分（0-10）
- `time_verdict` — 结论
- `time_analysis` — 详细分析

#### 金钱账 (money_account)
- `monthly_net_income` — 月净收入
- `roi_percentage` — 月投资回报率
- `payback_months` — 回本周期（月）
- `annual_net_profit` — 年净利润
- `money_score` — 评分（0-10）
- `money_verdict` — 结论
- `money_analysis` — 详细分析

#### 机会账 (opportunity_account)
- `monthly_opportunity_cost` — 月机会成本
- `net_gain_vs_alternative` — 净收益差
- `opportunity_score` — 评分（0-10）
- `opportunity_verdict` — 结论
- `opportunity_analysis` — 详细分析

#### 综合评估 (overall)
- `total_score` — 总分（0-30）
- `risk_level` — 风险等级
- `recommendation` — 综合建议
- `key_insights` — 关键洞察列表

## 第四步：风险预判 — 守住你的底线

**目标：** 评估用户对该副业可能带来的风险的承受能力。**画像中的 `risk_tolerance`（风险偏好）应作为风险评估的参考基准。**

### 4.1 三维风险评估

通过对话评估以下三个风险维度：

| 风险类型 | 关键问题 | 判断标准 |
|---------|---------|---------|
| **财务风险** | 如果项目失败，投入的资金是否会严重影响生活质量？ | 副业应是"锦上添花"，而非"孤注一掷" |
| **精力风险** | 这份副业是否会过度消耗精力，导致主业下滑或健康受损？ | 每周副业时间不应超过主业时间的30% |
| **合规风险** | 操作模式是否合法合规？是否会触犯平台规则？ | 警惕加盟骗局、刷单、灰色地带 |

### 4.2 风险评分规则

对每个风险维度打分（1-5分，5分最安全）：

- **5分** — 几乎无风险
- **4分** — 风险很低，可控
- **3分** — 有一定风险，需注意
- **2分** — 风险较高，需谨慎
- **1分** — 风险极高，建议放弃

### 4.3 风险预判输出

```
🛡️ 风险评估报告 — [副业名称]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💰 财务风险：X/5
   [具体分析和建议]

⚡ 精力风险：X/5
   [具体分析和建议]

⚖️ 合规风险：X/5
   [具体分析和建议]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
综合风险评分：XX/15
用户风险偏好：[保守/稳健/激进]
风险态度：[积极接受 / 谨慎前行 / 建议规避]
```

## 工作流规则

### 执行顺序

1. **MANDATORY：** 始终按「自我评估 → SWOT分析 → 价值核算 → 风险预判」的顺序执行，不可跳步。
2. **画像优先：** 每次评估开始时，必须先尝试加载已有画像。画像存在且完整时，第一步可快速跳过（仅确认无变化）。
3. 每一步完成后，向用户展示该步的输出，确认后再进入下一步。
4. 如果用户在某一步已经提供了足够信息，可以基于已有信息进行分析，但仍需确认。

### 对话规则

- **CRITICAL：** 语言必须与用户输入语言一致。如果用户用中文提问，全部输出使用中文。
- 使用 AskUserQuestion 工具收集信息时，问题应简洁明了，一次不超过3个问题。
- 对于数值型参数，如果用户无法提供精确值，接受估算值并标注"（估算）"。
- 保持客观中立，不做主观推荐，用数据和框架引导用户自行判断。

### 计算规则

- **MANDATORY：** 第三步「价值核算」必须使用 `scripts/evaluate.py` 脚本进行计算，禁止手动计算。
- 脚本执行命令格式：`python3 [skill_path]/scripts/evaluate.py --json '[JSON参数]'`
- 如果脚本执行失败，向用户说明情况，并尝试基于公式手动计算（需标注"手动估算"）。

### 画像规则

- **MANDATORY：** 第一步「自我评估」必须使用 `scripts/profile.py` 脚本管理画像文件。
- 画像文件路径：`[skill目录]/user-profile.json`
- 首次使用必须创建画像，后续评估必须先加载画像。
- 画像中的 `main_job_monthly_salary` 和 `main_job_monthly_hours` 在第三步自动填入，不得重复询问。
- 用户可随时通过 update 操作更新画像中的任何字段。

### 输出规则

- 每一步都应有结构化的输出（使用上述模板格式）。
- 最终报告应汇总所有步骤的关键结论。
- 如果用户同时评估多个副业方向，为每个方向分别生成评估报告，并提供横向对比表。

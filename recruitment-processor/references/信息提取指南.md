## 概述
本指南说明如何从招聘文档中准确提取关键信息，包括文本和图片内容的信息整合。

## 关键信息字段

### 1. 职位信息
**职位名称**
- 提取位置：文档标题、正文开头、职位描述段落
- 提取规则：选取最正式、最完整的职位名称
- 示例：高级Java开发工程师、产品经理（P5级）

**公司名称**
- 提取位置：文档标题、公司介绍段落、联系方式区域
- 提取规则：提取公司全称，如有简称一并记录
- 示例：阿里巴巴（中国）有限公司（简称阿里巴巴）

### 2. 薪资待遇
**薪资范围**
- 提取位置：薪资福利章节、职位描述开头
- 提取规则：
  - 优先提取明确的数值范围（如：15-25K/月）
  - 如为年薪，转换为"万/年"格式
  - 如为面议，标注"面议"
  - 如仅提下限（如"15K+"），标注为"15K+"
- 格式要求：数值范围+单位（如：15-25K/月，年薪30-40万）

**其他福利**
- 提取位置：薪资福利章节
- 提取规则：提取主要福利（五险一金、年终奖、带薪年假等）
- 示例：五险一金、年终奖、带薪年假、餐补、交通补贴

### 3. 时间信息
**截止时间**
- 提取位置：职位描述开头、招聘信息末尾
- 提取规则：
  - 提取明确的截止日期
  - 如未明确，标注"未提及"
  - 如为"招满即止"，标注"招满即止"
- 格式要求：YYYY-MM-DD

**发布时间**
- 提取位置：文档开头、元数据区域
- 提取规则：提取文档的发布日期或创建日期
- 格式要求：YYYY-MM-DD

### 4. 企业信息
**企业性质**
- 提取位置：公司介绍章节、职位描述中
- 提取规则：根据关键词判断
  - 国企关键词：国有、国资委、事业单位
  - 民企关键词：民营、私营、有限责任公司
  - 外企关键词：外资、合资、外企
  - 上市公司关键词：上市公司、上市企业
- 如无法判断，标注"未明确"

**工作地点**
- 提取位置：职位描述开头、工作要求章节
- 提取规则：提取城市和具体区域（如有）
- 格式要求：城市（区域），如：北京（朝阳区）

**公司规模**
- 提取位置：公司介绍章节
- 提取规则：提取人数规模或营收规模
- 示例：100-500人、500人以上、营收10亿+

**行业领域**
- 提取位置：公司介绍章节
- 提取规则：提取所属行业
- 示例：互联网、金融、教育、医疗

### 5. 任职要求
**学历要求**
- 提取位置：任职要求章节
- 提取规则：提取最低学历要求
- 可选值：博士、硕士、本科、大专、高中、不限
- 如为"本科及以上"，标注为"本科及以上"

**工作经验**
- 提取位置：任职要求章节
- 提取规则：
  - 提取明确的年限要求（如：1-3年、3-5年）
  - 如为"应届生可"，标注为"应届生可"
  - 如为"有相关经验"，标注为"有相关经验"
- 格式要求：明确的年限或描述性文字

**专业技能**
- 提取位置：任职要求章节
- 提取规则：提取关键技能列表
- 示例：Java、Spring Boot、MySQL、Redis、微服务

### 6. 岗位详情
**岗位职责**
- 提取位置：岗位职责章节
- 提取规则：提取核心职责要点，不超过5条
- 提取策略：选择最重要的、最能体现岗位价值的职责

**任职要求**
- 提取位置：任职要求章节
- 提取规则：提取核心要求要点，不超过5条
- 提取策略：选择硬性要求和核心能力要求

## 提取规则

### 图片内容识别
1. **识别流程**：
   - 逐一识别文档中的所有图片
   - 提取图片中的所有文字信息
   - 将图片文字与对应文本内容进行关联

2. **图片类型处理**：
   - 职位描述图片：重点识别薪资、要求等关键信息
   - 公司架构图片：识别部门、团队规模等信息
   - 薪资结构图片：识别具体薪资构成和福利
   - 其他图片：根据内容提取相关信息

3. **信息整合规则**：
   - 文本和图片信息一致：以文本为主，图片为补充
   - 文本和图片信息冲突：优先采用图片信息（图片通常更直观）
   - 图片信息更详细：以图片信息为准，补充文本信息

### 信息优先级
1. **明确性优先**：明确的数值或描述优于模糊描述
2. **官方性优先**：正式表述优于非正式表述
3. **完整性优先**：完整的信息优于部分信息

## 特殊情况处理

### 情况1：信息缺失
- 文档中未明确提及某项信息
- 处理方式：标注"未提及"
- 不允许主观推断或猜测

### 情况2：信息模糊
- 文档中描述模糊或不完整
- 处理方式：
  - 如能合理推断，标注推断结果并在括号内注明"推断"
  - 如无法合理推断，标注"不明确"
- 示例：薪资"面议"，标注"面议"

### 情况3：信息冲突
- 文本和图片信息冲突
- 处理方式：优先采用图片信息，并标注"信息来源：图片"

### 情况4：多值信息
- 某项信息有多个值
- 处理方式：用"或"连接所有值
- 示例：工作地点"北京或上海"

### 情况5：范围信息
- 某项信息为范围值
- 处理方式：保持原格式
- 示例：工作经验"1-3年"

## 示例

### 示例1：标准信息提取
**原始文档**：
```
# 高级Java开发工程师

公司：阿里巴巴
薪资：25-35K/月
学历：本科及以上
经验：3-5年
```

**提取结果**：
```
职位名称：高级Java开发工程师
公司名称：阿里巴巴
薪资待遇：25-35K/月
学历要求：本科及以上
工作经验：3-5年
```

### 示例2：图片内容识别
**文档内容**：
```
# 产品经理招聘

![职位详情](salary.png)

公司：腾讯
```

**图片salary.png内容**：
```
薪资范围：20-30K/月
年终奖：3-6个月
```

**提取结果**：
```
职位名称：产品经理
公司名称：腾讯
薪资待遇：20-30K/月（信息来源：图片）
其他福利：年终奖3-6个月（信息来源：图片）
```

### 示例3：信息整合
**文档内容**：
```
# 后端开发工程师

![薪资信息](img1.png)
![福利待遇](img2.png)

基本工资：18K
年终奖：3个月
```

**图片img1.png内容**：20-25K/月
**图片img2.png内容**：五险一金、餐补、交通补贴

**提取结果**：
```
职位名称：后端开发工程师
薪资待遇：20-25K/月（优先采用图片信息）
其他福利：年终奖3个月、五险一金、餐补、交通补贴
```

### 示例4：缺失信息处理
**原始文档**：
```
# UI设计师

公司：字节跳动
学历：本科
```

**提取结果**：
```
职位名称：UI设计师
公司名称：字节跳动
学历要求：本科
薪资待遇：未提及
工作经验：未提及
企业性质：未明确
```

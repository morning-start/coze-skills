# 操做手册

用户可能提出复杂操做，要拆解为原子操做进行。

## 操做原则

- 每个技能要分开提交处理
- 所有操做要尽可能拆分为原子操作，按照原子操作进行
- 最后推送到云端要作为单独步骤，不能和提交git同时进行

## 原子操做

新技能处理流程

1. 所有新技能要确定版本，如果没有则设置为1.0.0
2. 根据技能介绍完成git comment
3. 每个技能要分开提交git，不推送

技能修改处理流程

1. 修改后要能够根据变化自适应更新版本号
2. 根据技能变动完成git comment
3. 每个技能要分开提交git，不推送

更新发布版本的操做流程

1. 要根据变动内容自适应更新版本号
2. 根据技能变动完成 CHANGELOG 与 README 的撰写
3. 根据技能变动完成git comment
4. 自动提交git与tag，不推送

## 常见场景

### 更具项目变动自行处理

这种场景要注意区分技能的变动，

1. 分析技能的变动，确定是否需要更新版本号
2. 根据技能变动情况选择原子操作进行处理

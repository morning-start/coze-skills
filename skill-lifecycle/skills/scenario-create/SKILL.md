# 场景：创建新技能

## 适用场景

从零开始创建新技能，包括设计、开发、测试、发布四个阶段。

---

## 快速决策

```
新技能复杂度？
│
├─ 单一功能，正文 < 300行
│   └─ 用简单技能结构（单文件 SKILL.md）
│
└─ 多模块/协作/正文 ≥ 300行
    └─ 用复杂技能结构（SKILL.md + references/）
```

---

## 第一步：设计阶段

### 准入条件

需求已明确：核心问题、目标用户、触发场景

### 操作

**1. 确定技能类型**

| 维度 | 简单技能 | 复杂技能 |
|------|---------|---------|
| 功能数量 | 单一功能 | 多个子功能 |
| 正文行数 | < 300 行 | ≥ 300 行 |
| 配置复杂度 | 简单/无 | 复杂 |
| 依赖关系 | 无/简单 | 需要编排 |

**2. 初始化元信息**

```yaml
---
name: <skill-name>           # 小写+连字符
version: v0.1.0              # 开发版本
author: <作者>
description: <100-150字符>
tags: [tag1, tag2, tag3]    # 至少3个标签
---
```

**3. 创建目录**

```bash
mkdir <skill-name>
cd <skill-name>
touch SKILL.md
```

### 准出

- [ ] 技能类型已确定
- [ ] 元信息已初始化
- [ ] 目录已创建

---

## 第二步：开发阶段

### 准入条件

设计阶段准出条件全部满足

### 简单技能结构

```markdown
## 任务目标
- 本 Skill 用于: <一句话>
- 核心能力: <要点>
- 触发条件: <何时使用>

## 操作步骤
1. <步骤1>
2. <步骤2>

## 使用示例
<完整示例>

## 注意事项
<注意点>
```

### 复杂技能结构

**SKILL.md**（概览+索引）:
```markdown
# <技能标题>

## 任务目标
<一句话说明>

## 适用场景
<典型使用场景>

## 整体流程
<高层次流程图或步骤>

## 模块划分
- 模块1: <职责>
- 模块2: <职责>

## 子文档索引
- 实现: [references/implementation.md](references/implementation.md)
- 示例: [references/examples.md](references/examples.md)
```

### 准出

- [ ] 前言区字段完整
- [ ] 正文包含必需章节
- [ ] 示例完整可执行

---

## 第三步：测试阶段

### 准入条件

开发阶段准出条件全部满足

### 标准化检验

```bash
# 1. 检查前言区
head -10 SKILL.md

# 2. 检查描述长度（100-150字符）
grep "description:" SKILL.md | wc -c

# 3. 统计正文行数（< 500行）
tail -n +6 SKILL.md | wc -l

# 4. 检查标签数量（≥ 3个）
grep "tags:" SKILL.md
```

### 质量检查

- [ ] name 符合规范（小写+连字符）
- [ ] version 格式正确（v主.次.补丁）
- [ ] description 长度符合要求
- [ ] tags 至少 3 个
- [ ] 必需章节完整
- [ ] 示例可执行

---

## 第四步：发布阶段

### 准入条件

测试阶段准出条件全部满足

### 操作

```bash
# 更新版本号
# v0.1.0 → v1.0.0

git add .
git commit -m "feat(<skill-name>): 添加新技能"
git tag -a v1.0.0 -m "Release v1.0.0: <技能名称> 初始版本"
```

### 准出

- [ ] 版本号已更新（v1.0.0）
- [ ] Git commit 已创建
- [ ] Git tag 已创建

---

## 快速执行清单

```
□ 1. 设计
  □ 确定技能类型（简单/复杂）
  □ 初始化元信息
  □ 创建目录

□ 2. 开发
  □ 编写前言区
  □ 编写正文（简单: 单文件 / 复杂: 分层）
  □ 编写示例

□ 3. 测试
  □ 执行标准化检验
  □ 执行质量检查

□ 4. 发布
  □ 更新版本号
  □ Git 操作
```

---

## 常见问题

**Q: description 长度不足？**
- 补充使用场景说明
- 添加目标用户描述
- 扩展能力范围描述

**Q: 正文超过 500 行？**
- 使用复杂技能文档结构
- SKILL.md 仅作为概览
- 详细内容拆分到 references/

# 场景：创建新技能

## 核心流程

```
┌─────────────────────────────────────────────────────────────┐
│  技能创建核心流程                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ① 设计 ──→ ② 开发 ──→ ③ 测试 ──→ ④ 发布                  │
│    │           │           │           │                  │
│    ▼           ▼           ▼           ▼                  │
│  [元信息]   [SKILL.md]  [质量验收]  [版本标签]              │
│  [能力定义] [正文编写] [检查清单] [Git Tag]                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 关键节点速查

| 节点       | 准入条件 | 核心产出          | 准出条件     |
| ---------- | -------- | ----------------- | ------------ |
| **① 设计** | 需求明确 | 元信息 + 能力定义 | 设计文档完成 |
| **② 开发** | 设计通过 | SKILL.md          | 正文编写完成 |
| **③ 测试** | 开发完成 | 质量报告          | 检查清单通过 |
| **④ 发布** | 测试通过 | 版本标签          | Git 已推送   |

---

## 第一步：设计阶段

### 准入条件

✅ 需求已明确（核心问题、目标用户、使用场景）

### 核心操作

#### 1. 初始化元信息

```yaml
---
name: [skill-name] # 小写字母+连字符，不以 -skill 结尾
version: v0.1.0 # 开发版本
author: skill-lifecycle
description: [100-150字符描述]
tags: [tag1, tag2, tag3] # 至少 3 个标签
---
```

#### 2. 定义核心能力

```markdown
## 任务目标

- **本 Skill 用于**：[一句话说明技能用途]
- **核心能力**：
  - 能力 1：[说明]
  - 能力 2：[说明]
- **触发条件**：[何时使用]
```

#### 3. 创建目录结构

```bash
mkdir [skill-name]
cd [skill-name]
touch SKILL.md
```

### 准出条件

- [ ] 元信息完整（name、version、author、description、tags）
- [ ] 核心能力定义清晰
- [ ] 目录结构已创建

---

## 第二步：开发阶段

### 准入条件

✅ 设计阶段准出条件全部满足

### 核心操作

#### 1. 编写前言区

```yaml
---
name: [skill-name]
version: v0.1.0
author: skill-lifecycle
description: [100-150字符，清晰说明技能用途和场景]
tags: [tag1, tag2, tag3]
---
```

#### 2. 编写正文结构

```markdown
# [技能标题]

## 任务目标

- 本 Skill 用于：...
- 核心能力：...
- 触发条件：...

## 操作步骤

1. [步骤1]
2. [步骤2]
3. [步骤3]

## 使用示例

### 示例 1：[场景描述]

...

## 注意事项

- [注意点1]
- [注意点2]
```

#### 3. 编写使用示例

```markdown
## 使用示例

### 示例 1：[场景名称]

**输入**：
```

[输入数据/参数]

```

**操作**：
1. [操作步骤]
2. [操作步骤]

**输出**：
```

[预期输出]

```
```

### 准出条件

- [ ] 前言区字段完整
- [ ] 正文包含所有必需章节（任务目标、操作步骤、使用示例、注意事项）
- [ ] 示例完整可执行
- [ ] 正文行数 < 500 行

---

## 第三步：测试阶段

### 准入条件

✅ 开发阶段准出条件全部满足

### 核心操作

#### 1. 标准化检验

```bash
# 检查 1: 元信息完整性
head -10 SKILL.md

# 检查 2: description 长度（100-150字符）
grep "description:" SKILL.md

# 检查 3: 正文行数（< 500行）
wc -l SKILL.md

# 检查 4: tags 数量（≥ 3个）
grep "tags:" SKILL.md
```

#### 2. 质量检查清单

| 检查项      | 标准                                   | 结果 |
| ----------- | -------------------------------------- | ---- |
| name        | 小写字母+连字符，无 -skill 后缀        | □    |
| version     | vMajor.Minor.Patch 格式                | □    |
| description | 100-150 字符                           | □    |
| tags        | ≥ 3 个标签                             | □    |
| 正文结构    | 包含任务目标、操作步骤、示例、注意事项 | □    |
| 示例完整性  | 输入→操作→输出 完整                    | □    |

#### 3. 内容一致性检查

- [ ] 能力描述与操作步骤一致
- [ ] 示例与能力描述匹配
- [ ] 注意事项覆盖潜在问题

### 准出条件

- [ ] 所有标准化检验通过
- [ ] 所有质量检查通过
- [ ] 无阻塞性问题

---

## 第四步：发布阶段

### 准入条件

✅ 测试阶段准出条件全部满足

### 核心操作

#### 1. 准备发布

```bash
# 更新版本号（开发版 → 正式版）
# v0.1.0 → v1.0.0

git add .
git commit -m "feat([skill-name]): 添加新技能"
```

#### 2. 创建版本标签

```bash
git tag -a v1.0.0 -m "Release v1.0.0: [技能名称] 初始版本"

# 推送（不包含 tag，等待确认）
git push origin master
```

#### 3. 推送验证

```bash
# 确认推送成功
git log --oneline -1

# 确认状态
git status
```

### 准出条件

- [ ] 版本号已更新（v1.0.0）
- [ ] Git commit 已创建
- [ ] Git tag 已创建
- [ ] 代码已推送

## 快速执行清单

### 创建新技能时执行

```
□ 1. 设计阶段
  □ 明确需求（核心问题、目标用户、触发场景）
  □ 初始化元信息
  □ 定义核心能力
  □ 创建目录结构

□ 2. 开发阶段
  □ 编写前言区（name、version、author、description、tags）
  □ 编写正文（任务目标、操作步骤、示例、注意事项）
  □ 编写使用示例（输入→操作→输出）

□ 3. 测试阶段
  □ 执行标准化检验（4 项检查）
  □ 执行质量检查清单（6 项检查）
  □ 内容一致性检查（3 项检查）

□ 4. 发布阶段
  □ 更新版本号（v0.1.0 → v1.0.0）
  □ Git add + commit
  □ Git tag 创建
  □ Git push
```

## 常见问题

### Q1: description 长度不足 100 字符？

- 补充使用场景说明
- 添加目标用户描述
- 扩展能力范围描述

### Q2: 需要支持复杂功能？

- 使用复杂技能文档结构
- SKILL.md 仅作为概览
- 详细内容拆分到 references/

### Q3: 发现设计问题？

- 立即停止当前阶段
- 回退到设计阶段
- 重新完成设计后再继续

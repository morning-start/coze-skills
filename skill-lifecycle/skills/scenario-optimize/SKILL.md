# 场景：优化技能

## 适用场景

提升现有技能质量，包括结构优化、描述优化、示例补充。

---

## 快速决策

```
质量问题类型？
│
├─ 职责过多（>5个核心能力）
│   └─ 用原子化拆分策略
│
├─ 描述不规范/不清晰
│   └─ 用描述优化策略
│
├─ 示例不足（<2个）
│   └─ 用示例补充策略
│
└─ 接口不清晰
    └─ 用标准化策略
```

---

## 第一步：质量问题诊断

### 常见问题识别

| 问题类型 | 症状 | 优化策略 |
|---------|------|---------|
| 职责过多 | 一个技能 >5 个能力 | 原子化拆分 |
| 描述不清 | description 不符合规范 | 描述优化 |
| 示例不足 | 少于 2 个示例 | 示例补充 |
| 接口混乱 | 输入输出不清晰 | 标准化 |

### 优化目标定义

```yaml
优化目标:
  问题类型: <识别的问题>
  优化策略: <选择的策略>
  预期改进: <具体改进点>
```

---

## 第二步：执行优化

### 策略 1: 原子化拆分

适用：技能职责过多（>5 个核心能力）

**拆分原则**:
- 每个技能单一职责
- 技能间松耦合
- 可独立使用

**示例**:
```yaml
# 拆分前
data-processor: [读取, 清洗, 分析, 验证, 导出]

# 拆分后
data-reader: [读取]
data-cleaner: [清洗]
data-analyzer: [分析]
data-exporter: [导出]
```

### 策略 2: 描述优化

适用：description 不符合规范（100-150字符）

**优化方法**:
- 补充使用场景
- 添加目标用户
- 扩展能力范围

### 策略 3: 示例补充

适用：示例少于 2 个

**补充要求**:
- 每个示例包含输入→操作→输出
- 覆盖主要使用场景
- 示例可复制执行

### 策略 4: 标准化接口

适用：输入输出不清晰

**标准化内容**:
- 输入参数格式
- 输出结果格式
- 错误处理

---

## 第三步：验证效果

### 质量检查

- [ ] description 符合规范（100-150字符）
- [ ] 示例完整可执行
- [ ] 结构清晰合理
- [ ] 无引入新问题

|------|-------|-------|

| 示例完整性 | - | - |
| 结构合理性 | - | - |

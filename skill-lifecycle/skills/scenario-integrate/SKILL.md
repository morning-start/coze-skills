# 场景：整合技能

## 适用场景

将多个相关技能整合为一个统一的能力集合。

---

## 快速决策

```
整合模式选择：
│
├─ 流水线处理（数据依次传递）
│   └─ 用顺序模式
│
├─ 多任务同时执行
│   └─ 用并行模式
│
└─ 分层处理（外层控制内层）
    └─ 用嵌套模式
```

---

## 第一步：分析源技能

### 操作

**1. 列出源技能**

```yaml
源技能:
  skill-a:
    capabilities: [能力1, 能力2]
  skill-b:
    capabilities: [能力3, 能力4]
```

**2. 检查兼容性**

- [ ] 命名无冲突
- [ ] 接口可统一
- [ ] 数据格式可转换

**3. 选择整合模式**

| 模式 | 说明 | 适用场景 | 示例 |
|------|------|---------|------|
| 顺序 | 技能按顺序执行 | 流水线处理 | 清洗→分析→报告 |
| 并行 | 技能同时执行 | 独立任务 | 多条件验证 |
| 嵌套 | 一个技能调用另一个 | 分层处理 | 外层控制+内层执行 |

---

## 第二步：设计整合

### 操作

**1. 设计整合架构**

```yaml
整合设计:
  名称: <整合技能名>
  模式: <顺序/并行/嵌套>
  组件:
    - <skill-a>
    - <skill-b>
```

**2. 设计数据流**

```
输入 → [skill-a] → [skill-b] → 输出
```

---

## 第三步：执行整合

### 开发阶段

**准入**: 整合设计完成

**操作**:

1. 创建整合技能目录
2. 编写 SKILL.md

   ```yaml
   ---
   name: <整合技能名>
   version: v1.0.0
   description: <100-150字符>
   tags: [<相关标签>]
   ---
   ```

3. 定义整合流程

   ```markdown
   ## 任务目标
   - 本 Skill 用于: <一句话>
   - 能力包含:
     - 调用 <skill-a>
     - 调用 <skill-b>
   - 触发条件: <何时使用>

   ## 操作步骤
   1. 调用 <skill-a>
   2. 调用 <skill-b>
   ```

### 测试阶段

**准入**: 开发阶段完成

**验证**:
- [ ] 接口契约正确
- [ ] 数据传递正确
- [ ] 错误处理完善

### 发布阶段

**准入**: 测试阶段完成

```bash
git add .
git commit -m "feat(<整合技能>): 整合 <skill-a> 和 <skill-b>"
git tag -a v1.0.0 -m "Release v1.0.0"
```

---

## 第四步：验收

### 整合质量检查

- [ ] 整合模式选择合理
- [ ] 数据流设计清晰
- [ ] 接口契约明确
- [ ] 依赖声明完整

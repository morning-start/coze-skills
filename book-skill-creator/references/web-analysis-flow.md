# 智能网站分析流程

## 概述

本详细说明智能网站分析模式（模式 A）的完整流程，包括网站内容读取、迭代学习、技能规划等步骤。

## 完整流程

```
网站 URL → 内容读取 → 技术特点分析 → 知识规划 → 迭代学习 → 技能规划 → 并行生成
```

## 步骤 A.1：网站内容读取与初步分析

### 1. 获取网站 URL
- 用户提供技术网站/官方文档 URL
- 支持单个或多个 URL 输入
- 验证 URL 可访问性

### 2. 读取网站内容
- 使用 **WebFetch** 工具获取网页内容
- 使用 **Read** 工具读取本地文档（如有）
- 提取文本、代码示例、链接等信息

### 3. 技术特点初步提取
- 识别技术名称和版本
- 提取核心功能描述
- 识别应用场景
- 记录关键术语和概念

### 4. 输出初步分析报告

```markdown
## 技术初步分析

### 技术信息
- **名称**：{技术名称}
- **版本**：{版本信息}
- **官网**：{URL}
- **类型**：{框架/库/语言/工具}

### 核心功能
1. {功能点 1}
2. {功能点 2}
3. {功能点 3}

### 应用场景
- {场景 1}
- {场景 2}

### 已识别的关键概念
| 概念名称 | 简要描述 | 来源链接 |
|---------|---------|---------|
| {概念 1} | {描述} | {URL} |
| {概念 2} | {描述} | {URL} |
```

## 步骤 A.2：生成知识规划

### 1. 分析知识缺口
- 基于初步分析，识别需要深入了解的内容
- 确定技术栈的广度和深度
- 评估现有信息的完整性

### 2. 生成知识规划清单
- 列出需要了解的主题
- 为每个主题提供相关链接
- 标注优先级（高/中/低）

### 3. 输出知识规划

```markdown
## 知识规划

### 高优先级
| 主题 | 说明 | 相关链接 |
|-----|------|---------|
| {主题 1} | {说明} | [链接 1]({URL}), [链接 2]({URL}) |

### 中优先级
| 主题 | 说明 | 相关链接 |
|-----|------|---------|
| {主题 2} | {说明} | [链接]({URL}) |

### 低优先级
| 主题 | 说明 | 相关链接 |
|-----|------|---------|
| {主题 3} | {说明} | [链接]({URL}) |
```

## 步骤 A.3：迭代式学习循环

### 1. 执行搜索获取更多内容
对每个高优先级主题使用 **search** 工具：
- `"{技术名} {主题} 官方文档"`
- `"{技术名} {主题} best practices"`
- `"{技术名} {主题} tutorial"`

### 2. 深度分析和补充
- 整合新获取的内容
- 补充技术特点分析
- 更新知识规划

### 3. 判断是否继续学习
- 评估信息完整性（目标：80%+ 覆盖率）
- 检查是否还有重要主题未覆盖
- 判断是否需要进一步迭代

### 4. 迭代决策

```markdown
## 迭代学习记录

### 第 {N} 轮迭代
- **搜索主题**：{主题列表}
- **新增内容**：
  - {新增知识点 1}
  - {新增知识点 2}
- **信息完整性**：{X}%
- **继续迭代**：是/否
- **原因**：{说明}
```

### 5. 循环执行
- 如果"继续迭代"= 是：返回步骤 A.2
- 如果"继续迭代"= 否：进入步骤 A.4

## 步骤 A.4：技能规划

### 1. 技术模块识别
基于完整分析，识别独立的技术模块：
- 核心概念和原理
- API/函数/配置参考
- 代码示例
- 最佳实践
- 记录模块间的依赖关系

### 2. 生成技能拆分计划

```markdown
# 技能拆分计划：{技术名称}

## 计划概述
- **源网站**：{URL}
- **分析轮次**：{N} 轮
- **计划生成时间**：{日期}
- **预计生成技能数**：{N} 个

## 技能清单

### 技能 1：{skill-name-1}
- **对应模块**：{模块 1, 模块 2}
- **技能定位**：{基础/核心/高级}
- **核心内容**：{内容点 1}, {内容点 2}
- **依赖技能**：{无/技能 X}
- **预计复杂度**：{低/中/高}

### 技能 2：{skill-name-2}
...

## 执行策略

### 并行执行组
- **第 1 组**（无依赖，可并行）：{skill-name-1}
- **第 2 组**（依赖第 1 组，可并行）：{skill-name-2}, {skill-name-3}
- **第 3 组**（依赖前两组，可并行）：{skill-name-4}

### 生成顺序建议
1. 先生成基础技能（无依赖）
2. 再并行生成依赖基础技能的核心技能
3. 最后生成高级技能
```

### 3. 与用户确认计划
- 展示技能拆分计划
- 根据用户反馈调整
- 确认最终执行方案

## 质量检查清单

- [ ] 网站 URL 可访问性验证通过
- [ ] 核心功能提取完整（至少 3 个）
- [ ] 关键概念提取完整（至少 5 个）
- [ ] 知识规划清单完整（高/中/低优先级分明）
- [ ] 至少执行 1 轮迭代学习
- [ ] 信息完整性达到 80% 以上
- [ ] 技能拆分合理（每个技能职责单一）
- [ ] 依赖关系分析正确

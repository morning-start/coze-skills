# Meta Skill Creator

## 任务目标
- 本 Skill 用于: 创建、管理、优化和演化其他技能，提供从基础定义到高阶组合的全生命周期管理
- 能力包含:
  - **技能创建**: 从零构建标准化技能定义
  - **技能管理**: 增加、修改、更新、版本控制
  - **技能整合**: 将2+个技能合并为1个新技能
  - **技能拆分**: 将1个技能拆分为1+个独立技能
  - **工作流生成**: 基于多技能生成标准化 WORKFLOW.md
  - **自省分析**: 能力边界分析、优化建议
- 触发条件: 用户需要创建新技能、管理现有技能、整合/拆分技能、生成工作流或管理技能版本

## 前置准备
- 了解 Skill 标准规范: 参考 [references/skill-standards.md](references/skill-standards.md) 掌握命名规范、目录结构和格式要求
- 了解技能组合模式: 参考 [references/composition-patterns.md](references/composition-patterns.md) 掌握四种标准组合模式

## 操作步骤

### 一、技能创建流程
1. 需求分析与上下文理解
   - 分析用户目标、历史记录、约束条件
   - 结合 [references/skill-standards.md](references/skill-standards.md) 确定技能类型和规范
2. 生成技能定义
   - 按照标准格式生成 YAML 前言区
   - 编写 Markdown 正文结构（任务目标、前置准备、操作步骤、资源索引、注意事项）
3. 格式验证
   - 检查前言区字段完整性
   - 检查正文章节结构
   - 检查链接层级（禁止嵌套引用）

### 二、技能管理流程
1. **增加技能**
   - 分析新技能与现有技能的关系
   - 确定技能目录结构和命名
   - 生成完整的 SKILL.md 文件

2. **修改技能**
   - 读取现有技能定义
   - 分析修改需求的影响范围
   - 更新 SKILL.md 内容
   - 评估是否需要版本更新

3. **更新技能**
   - 对比当前版本与目标版本的差异
   - 识别新增/修改/删除的内容
   - 执行更新并记录变更

4. **版本控制**
   - 分析变更性质（major/minor/patch）
   - 更新版本号
   - 记录变更日志（changelog）
   - 语义化版本规则:
     - major: 破坏性变更（移除能力、接口改变）
     - minor: 新增功能（添加能力、不破坏兼容）
     - patch: 错误修复（Bug 修复、文档更新）

### 三、技能整合流程
1. 分析输入技能
   - 读取所有待整合技能的 SKILL.md
   - 提取每个技能的核心能力、操作步骤、资源索引
   - 识别能力重叠和互补关系

2. 设计整合策略
   - 确定新技能的核心目标
   - 规划能力融合方式（顺序/并行/嵌套/流水线）
   - 设计统一的输入输出接口

3. 生成整合后技能
   - 创建新的 SKILL.md
   - 合并并优化操作步骤
   - 整合资源索引
   - 确保版本从 1.0.0 开始

4. 验证整合效果
   - 检查能力完整性
   - 验证流程连贯性
   - 确认无重复或冲突

### 四、技能拆分流程
1. 分析源技能
   - 读取源技能 SKILL.md
   - 识别独立的能力模块
   - 评估拆分粒度和边界

2. 规划拆分方案
   - 确定拆分为几个子技能
   - 为每个子技能定义独立目标
   - 规划技能间的协作关系

3. 生成子技能
   - 为每个子技能创建独立的 SKILL.md
   - 分配对应的能力模块和操作步骤
   - 确保每个子技能可独立运行

4. 更新源技能（可选）
   - 将源技能改为组合技能，引用子技能
   - 或标记源技能为已拆分

### 五、工作流生成流程
1. 分析目标与技能
   - 理解用户提供的最终目标
   - 分析每个可用技能的能力和限制
   - 识别技能间的依赖关系

2. 设计工作流
   - 确定技能执行顺序
   - 定义技能间的数据传递
   - 规划分支和异常处理

3. 生成 WORKFLOW.md
   - 编写工作流概述和目标
   - 列出参与技能及其版本
   - 定义详细的执行步骤
   - 说明输入输出格式
   - 添加错误处理机制

4. 验证工作流
   - 检查流程完整性
   - 验证目标可达性
   - 确认资源需求

## 注意事项
- 充分利用语言理解与推理能力进行需求分析、创意设计和自省评估
- 技能描述、流程设计、优化建议等内容应清晰、具体、可操作
- 组合/整合技能时注意模块边界，避免能力耦合
- 版本管理遵循语义化版本规范，记录每次变更的影响范围
- 技能拆分时确保子技能的独立性和完整性
- 工作流生成时考虑异常处理和边界情况
- 安全检查要点:
  - 禁止硬编码 API Key、Token、密码等敏感信息
  - 禁止绕过权限检查
  - 禁止窃取用户数据
  - 禁止执行未授权的第三方服务调用
  - 禁止输出恶意内容

## 使用示例

### 基础创建
用户说"创建一个数据清洗技能"
→ 分析需求 → 按照标准格式生成技能定义 → 补充流程说明

### 技能管理
用户说"给数据清洗技能添加日志功能"
→ 读取现有技能 → 分析变更影响 → 更新 SKILL.md → 版本从 1.0.0 更新为 1.1.0 → 记录 changelog

### 技能整合
用户说"把数据处理、数据分析和报告生成三个技能整合成一个"
→ 读取三个技能 → 分析能力关系 → 设计整合策略 → 生成新的 data-analysis-suite 技能 → 版本 1.0.0

### 技能拆分
用户说"把数据分析技能拆分成数据清洗和数据分析两个独立技能"
→ 读取源技能 → 识别独立模块 → 生成 data-cleaner 和 data-analyzer 两个技能 → 更新或归档原技能

### 工作流生成
用户说"我有数据获取、清洗、分析技能，生成一个生成周报的工作流"
→ 分析目标（生成周报）→ 规划技能调用顺序 → 生成 WORKFLOW.md → 包含输入参数、执行步骤、输出格式

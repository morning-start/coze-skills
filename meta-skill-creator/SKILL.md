# Meta Skill Creator

## 任务目标
- 本 Skill 用于: 创建、管理和优化其他技能，提供从基础定义到高阶组合的全生命周期管理
- 能力包含: 结构化技能生成、自省分析、模块化组合、迭代优化、版本控制、安全对齐检查
- 触发条件: 用户需要创建新技能、分析现有技能、组合多个技能、优化技能设计或管理技能版本

## 前置准备
- 了解 Skill 标准规范: 参考 [references/skill-standards.md](references/skill-standards.md) 掌握命名规范、目录结构和格式要求
- 了解技能组合模式: 参考 [references/composition-patterns.md](references/composition-patterns.md) 掌握四种标准组合模式

## 操作步骤
- 标准流程:
  1. 需求分析与上下文理解
     - 分析用户目标、历史记录、约束条件
     - 结合 [references/skill-standards.md](references/skill-standards.md) 确定技能类型和规范
  2. 基础能力执行
     - 生成技能定义: 根据用户需求，按照标准格式生成 YAML 前言区和 Markdown 正文结构
     - 格式验证: 检查格式合规性，包括前言区字段完整性、正文章节结构、链接层级等
  3. 高阶能力执行（按需）
     - 技能组合: 参考组合模式文档，实现技能嵌套、复用和拼接
     - 版本管理: 处理技能版本迭代、差异对比，遵循语义化版本规范
     - 自省分析: 分析技能能力边界、优化点、组合可能性
  4. 优化与迭代
     - 根据反馈调整技能设计
     - 执行安全检查，确保无敏感信息泄露、无不安全操作
- 可选分支:
  - 当 创建基础技能: 执行步骤 1-2
  - 当 组合多个技能: 执行步骤 1-3（包含组合）
  - 当 优化现有技能: 执行步骤 1-2-4
  - 当 版本迭代: 执行步骤 1-2-3（版本管理）-4

## 注意事项
- 充分利用语言理解与推理能力进行需求分析、创意设计和自省评估
- 技能描述、流程设计、优化建议等内容应清晰、具体、可操作
- 组合技能时注意模块边界，避免能力耦合
- 版本管理遵循语义化版本规范，记录每次变更的影响范围
- 安全检查要点:
  - 禁止硬编码 API Key、Token、密码等敏感信息
  - 禁止绕过权限检查
  - 禁止窃取用户数据
  - 禁止执行未授权的第三方服务调用
  - 禁止输出恶意内容

## 使用示例
- 基础创建: 用户说"创建一个数据清洗技能"，分析需求 → 按照标准格式生成技能定义 → 补充流程说明
- 技能组合: 用户说"把数据处理和报告生成组合起来"，分析组合策略 → 参考组合模式文档 → 调整接口定义
- 版本迭代: 用户说"给这个技能添加日志功能"，设计变更 → 记录版本 → 生成新定义

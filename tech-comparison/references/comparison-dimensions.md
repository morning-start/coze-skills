## 前端框架

### 核心维度
- **学习曲线**：上手难度、文档质量、社区教程丰富度
- **性能**：运行时性能（渲染速度、更新效率）、内存占用
- **生态成熟度**：第三方库数量、质量、维护状态
- **社区活跃度**：GitHub Stars、Contributors、Issue响应速度
- **TypeScript支持**：类型定义完整性、开发体验
- **打包体积**：生产环境bundle大小、tree-shaking能力

### 可选维度
- **SSR/SSG支持**：服务端渲染、静态站点生成能力
- **开发体验**：调试工具、热更新、开发服务器性能
- **长期维护性**：版本规划、向后兼容性、企业采用情况
- **框架特色**：响应式系统（Vue）、虚拟DOM（React）、编译时优化（Svelte）

### 典型关键词与子生态示例
**主技术栈**：
- React: React.js、React生态
- Vue: Vue.js、Vue生态
- Svelte: Svelte生态
- Angular: Angular生态
- Solid: SolidJS、Solid生态

**子生态（需归为主技术栈，并额外考虑SSR/SSG维度）**：
- Next.js → 归为React，额外考虑SSR/SSG维度
- Nuxt.js → 归为Vue，额外考虑SSR/SSG维度
- Remix → 归为React，额外考虑SSR/SSG维度
- SvelteKit → 归为Svelte，额外考虑SSR/SSG维度
- Astro → 跨框架，独立分析

**移动端/跨平台（原生支持）**：
- React Native → 归为React，考虑跨平台支持维度
- Vue Native → 归为Vue，考虑跨平台支持维度
- NativeScript → 独立分析

### 不适用维度说明
- 部署方案相关维度（如跨平台支持）不适用，除非框架原生支持（如React Native）

## 后端语言/框架

### 核心维度
- **性能**：请求处理速度、并发能力、资源占用
- **开发效率**：开发速度、代码简洁性、开发工具支持
- **生态成熟度**：第三方库、中间件、插件生态
- **学习曲线**：语言难度、框架复杂性、文档质量
- **适用场景**：Web API、微服务、数据处理、实时应用等

### 可选维度
- **类型系统**：静态类型支持（TS/Java）、动态类型（Python/Node.js）
- **并发模型**：多线程（Java/Go）、事件循环（Node.js）、协程（Python async）
- **社区活跃度**：GitHub活跃度、包管理器生态（npm/pip/maven）
- **长期维护性**：版本稳定性、企业采用情况、就业市场

### 典型关键词
**语言**：
- Python: Python生态
- JavaScript/TypeScript: Node.js生态
- Go: Go语言生态
- Java: Java生态
- Rust: Rust生态
- C#: .NET生态

**Python框架**：
- Django: 全栈框架、ORM、admin
- Flask: 微框架、轻量级、灵活
- FastAPI: 异步、高性能、自动文档

**Node.js框架**：
- Express: 灵活、中间件生态
- NestJS: 企业级、模块化、TypeScript原生
- Koa: 轻量级、中间件

**Go框架**：
- Gin: 高性能、路由
- Echo: 轻量级、中间件
- Fiber: 快速、类似Express

**Java框架**：
- Spring Boot: 企业级、生态丰富
- Micronaut: 轻量级、云原生

### 不适用维度说明
- 前端相关维度（如渲染性能）不适用

## 数据库

### 核心维度
- **数据模型**：关系型、文档型、键值、图、时序等
- **性能**：读写速度、查询性能、索引效率
- **扩展性**：垂直扩展、水平扩展、分片能力
- **一致性**：ACID支持、CAP定理权衡
- **适用场景**：事务型、分析型、实时性要求等

### 可选维度
- **查询能力**：SQL支持、查询语言复杂度、聚合能力
- **运维复杂度**：部署难度、监控、备份、故障恢复
- **社区活跃度**：文档质量、商业支持、企业采用
- **云服务支持**：托管服务可用性、成本、SLA

### 典型关键词
**关系型数据库（SQL）**：
- MySQL: 开源、流行
- PostgreSQL: 功能丰富、扩展性强
- SQLite: 嵌入式、零配置
- SQL Server: 企业级、Windows

**文档型数据库**：
- MongoDB: 文档存储、灵活模式
- Couchbase: 内存优先、高性能

**键值数据库**：
- Redis: 内存数据库、缓存
- DynamoDB: 无服务器、云原生
- Memcached: 简单缓存

**图数据库**：
- Neo4j: 原生图、关系查询
- ArangoDB: 多模型

**时序数据库**：
- InfluxDB: 时序数据、监控
- TimescaleDB: PostgreSQL扩展

**搜索引擎**：
- Elasticsearch: 全文搜索、分析
- Apache Solr: 企业级搜索

### 不适用维度说明
- 学习曲线维度通常不关键（SQL是通用标准）
- 打包体积不适用

## 部署方案

### 核心维度
- **性能**：运行时性能、资源占用（CPU/内存/磁盘）
- **打包体积**：安装包大小、依赖体积
- **跨平台支持**：Windows、macOS、Linux、移动端支持
- **开发体验**：调试能力、热更新、构建速度

### 可选维度
- **生态成熟度**：社区插件、集成方案、示例项目
- **学习曲线**：上手难度、文档质量、迁移成本
- **长期维护性**：版本稳定性、企业采用、更新频率
- **安全性与更新**：安全补丁、依赖更新、签名机制

### 典型关键词
**桌面端跨平台**：
- Electron: 基于Chromium、生态丰富、资源占用高
- Tauri: 基于WebView2、轻量级、Rust后端
- Neutralino: 极致轻量、依赖少
- Flutter: Google、跨平台、Dart语言

**移动端跨平台**：
- React Native: Meta、JavaScript、热更新
- Flutter: Google、Dart、性能优异
- Ionic: Web技术、多框架

**容器化**：
- Docker: 标准容器、生态成熟
- Podman: 无守护进程、安全

**无服务器**：
- Vercel: Next.js友好、边缘计算
- Netlify: 静态站点、CI/CD
- AWS Lambda: 云原生、事件驱动

### 不适用维度说明
- 纯打包工具（如Webpack）不涉及跨平台支持
- 静态站点生成器不涉及运行时性能

## 通用维度

### 适用于所有技术类型
- **长期维护性**：
  - 版本规划（是否有明确的LTS/稳定版本）
  - 向后兼容性（升级成本）
  - 企业采用情况（是否被大公司使用）
  - 维护团队稳定性（是否由公司或基金会维护）

- **学习曲线**：
  - 文档质量（完整性、清晰度、示例代码）
  - 社区教程（博客、视频、课程）
  - 上手难度（概念复杂度、配置复杂度）

- **社区活跃度**：
  - GitHub指标：Stars、Contributors、Commit频率
  - Stack Overflow：问题数量、回答速度
  - 社交媒体：讨论热度、更新频率

### 使用原则
- 根据技术类型选择核心维度作为必须对比项
- 可选维度根据具体情况灵活选择
- 不适用维度明确标注"不适用"
- 通用维度适用于所有技术类型
- 子生态技术需归为主技术栈，并额外考虑特有维度

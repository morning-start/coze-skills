# Skill Factory Analyzer - 技术内容分析器

## 技能职责边界

**本技能负责**：
- 读取并解析 URL 或文档内容
- 提取技术名称、版本、核心功能
- 识别关键概念和应用场景
- 评估信息完整度，决定是否需要迭代补充

**本技能不负责**：
- 技能拆分设计（由 planner 负责）
- 技能文件生成（由 generator 负责）
- 知识内容的深度学习（只做提取，不做内化）

---

## 输入判断

### 有效输入

| 输入类型 | 有效条件 |
|----------|----------|
| URL | 可访问（HTTP 200/301/302），返回技术文档内容 |
| 本地文档路径 | 文件存在，可读，内容 > 100 字符 |

### 无效输入处理

- URL 不可访问 → 返回错误："无法访问该 URL，请检查地址或提供文档内容"
- 文件不存在 → 返回错误："文件不存在，请检查路径"
- 内容 < 100 字符 → 返回错误："内容过少，无法进行分析"

---

## 核心逻辑

### 工作流程

```
输入 → 有效性验证 → 内容读取 → 结构化提取 → 完整度评估 → 输出
                                    ↓
                           完整度 < 80%？
                              ↓         ↓
                             是        否
                         迭代补充    输出报告
```

### 步骤详解

#### 步骤 1：内容读取

**URL 输入**：
1. 使用 `WebFetch` 获取网页内容
2. 验证返回内容非空

**文档输入**：
1. 使用 `Read` 读取文件内容
2. 验证内容可解析

#### 步骤 2：结构化提取

从内容中提取以下字段：

| 字段 | 提取目标 | 示例 |
|------|----------|------|
| 技术名称 | 标题/正文中的技术名 | Vue.js, React, TensorFlow |
| 版本信息 | 文档中的版本号 | v3.0, 2.0, 2024 |
| 类型分类 | 框架/库/语言/工具/平台 | 框架 |
| 核心功能 | 列表形式，3-5 个主要功能 | 组件系统、响应式数据、路由管理 |
| 应用场景 | 列表形式，2-4 个典型场景 | Web 开发、单页应用、移动端 |
| 关键概念 | 表格形式，术语+简要说明 | 虚拟 DOM、JSX、依赖注入 |
| 官方文档链接 | 文档中的链接 | /guide, /docs, /tutorial |

#### 步骤 3：完整度评估

**评估公式**：

```
完整度 = 已提取字段数 / 7 个必填字段 × 100%
```

**必填字段**（7个）：
1. 技术名称
2. 类型分类
3. 核心功能（至少 1 个）
4. 应用场景（至少 1 个）
5. 关键概念（至少 2 个）
6. 官方文档链接
7. 版本信息（可选，无则标注"未明确"）

#### 步骤 4：迭代补充（完整度 < 80% 时触发）

1. 识别信息缺口：哪些字段未提取或提取不充分
2. 执行搜索：对未提取的字段使用 `WebSearch` 补充
   - 搜索关键词："{技术名称} 官方文档 {缺口字段}"
3. 重新评估完整度
4. **最多迭代 2 次**，第 3 次迭代后无论完整度多少都输出当前结果

---

## 输出标准

### 合格报告

必须包含所有 7 个必填字段，且完整度 ≥ 80%。

### 输出格式

```markdown
## 技术分析报告

### 基本信息
- **技术名称**：[名称]
- **版本**：vX.X 或 未明确
- **类型**：框架/库/语言/工具/平台

### 核心功能
1. [功能 1]
2. [功能 2]
3. [功能 3]

### 应用场景
- [场景 1]
- [场景 2]

### 关键概念
| 概念 | 说明 |
|------|------|
| 概念1 | 说明1 |
| 概念2 | 说明2 |

### 参考链接
- [官方文档](URL)
- [快速入门](URL)

### 完整度评估
- **评估结果**：XX%
- **迭代次数**：X/2
- **未提取字段**：暂无 / [字段列表]
```

---

## 失败处理

| 失败场景 | 处理方式 |
|----------|----------|
| 输入 URL 无效 | 返回错误，不执行分析 |
| 输入文件不存在 | 返回错误，不执行分析 |
| 内容过少无法分析 | 返回错误，提供最低字数要求 |
| 迭代 2 次后完整度仍 < 80% | 输出当前报告，标注完整度和未提取字段，由 planner 判断是否继续 |
| 网页内容无法解析 | 尝试 fallback 方式（降级到 title 和 meta 提取），仍失败则返回错误 |

**输入 1**：`https://vuejs.org`

**输出 1**：
```markdown
## 技术分析报告

### 基本信息
- **技术名称**：Vue.js
- **版本**：3.4
- **类型**：框架

### 核心功能
1. 响应式数据绑定
2. 组件系统
3. 指令系统
4. 路由管理
5. 状态管理

### 应用场景
- 单页应用（SPA）
- 企业级应用
- 中小型项目

### 关键概念
| 概念 | 说明 |
|------|------|
| 响应式 | 数据变化自动更新视图 |
| 组件 | 可复用的 Vue 实例 |
| 指令 | v-bind, v-if, v-for 等 |
| Composition API | 组合式 API |

### 参考链接
- [官方文档](https://vuejs.org/guide/)
- [API 参考](https://vuejs.org/api/)

### 完整度评估
- **评估结果**：95%
- **迭代次数**：0/2
- **未提取字段**：暂无
```

## 概述

本指南详细说明如何使用智能网站分析模式从技术网站/官方文档生成技能拆分计划。

### 核心流程

```
网站 URL → 内容读取 → 技术特点分析 → 知识规划 → 迭代学习 → 技能规划
```

### 适用场景

- 用户提供技术官网 URL，需要快速了解该技术
- 需要从官方文档生成多个相关技能
- 需要系统性地学习新技术
- 需要为团队制定技术学习路径

## 网站内容读取

### 工具选择

| Read | 读取本地文件 | 下载的文档、本地笔记 |

#### 1. 首页分析
- 提取技术名称和版本
- 识别核心功能描述
- 获取快速开始指南
- 记录重要链接

#### 2. 导航结构分析
- 提取文档目录
- 识别章节组织方式
- 标记核心章节
- 记录 API 参考位置

#### 3. 内容提取
```python

    "tutorials": "教程和指南"

```

- **技术类型**：框架/库/语言/工具

- 性能特点

- 构建工具（Webpack、Vite 等）

## 知识规划

### 知识缺口分析

#### 1. 广度分析
识别技术的各个组成部分：
- 核心概念
- API 参考
- 工具链
- 生态系统
- 最佳实践

#### 2. 深度分析
评估每个部分需要了解的深度：
- **了解**：知道是什么
- **理解**：知道为什么
- **掌握**：能够熟练使用
- **精通**：能够优化和教学

#### 3. 优先级划分

| 优先级 | 标准 | 示例 |
|--------|------|------|
| 高 | 核心概念，必须掌握 | 响应式原理、组件系统 |
| 中 | 重要功能，经常使用 | 路由、状态管理 |
| 低 | 高级特性，偶尔使用 | 服务端渲染、插件开发 |

## 迭代式学习

### 迭代流程

```
第 1 轮：核心概念学习
  ↓
评估完整性（60%）
  ↓
第 2 轮：重要功能补充
  ↓
评估完整性（80%）
  ↓
第 3 轮：高级特性和最佳实践
  ↓
评估完整性（90%+）→ 完成
```

#### 搜索执行

1. **批量搜索**：对多个主题同时搜索
2. **结果筛选**：优先选择官方资源
3. **内容提取**：获取详细教程和示例
4. **信息整合**：补充到知识规划中

### 完整性评估

#### 评估指标

| 指标 | 权重 | 评估方法 |
|------|------|---------|
| 核心概念覆盖率 | 40% | 检查是否覆盖所有核心概念 |
| API 完整性 | 30% | 检查主要 API 是否了解 |
| 实战场景 | 20% | 检查是否有实际案例 |
| 最佳实践 | 10% | 检查是否了解最佳实践 |

#### 完整性计算公式

```
完整性 = (核心概念数 × 0.4 + API 覆盖数 × 0.3 + 
         实战场景数 × 0.2 + 最佳实践数 × 0.1) / 
         总知识点数 × 100%
```

## 技能规划

### 技能拆分原则

#### 1. 单一职责
每个技能只负责一个明确的技术模块

#### 2. 依赖清晰
技能间的依赖关系明确，避免循环依赖

#### 3. 难度递进
从基础到高级，循序渐进

#### 4. 实用导向
每个技能都对应实际使用场景

### 技能分类

| 分类 | 说明 | 示例 |
|------|------|------|
| 基础技能 | 核心概念和原理 | vue-core-skill |
| 核心技能 | 常用功能和 API | vue-router-skill |
| 高级技能 | 优化和扩展 | vue-ssr-skill |
| 工具技能 | 配套工具使用 | vue-devtools-skill |

## 最佳实践

### 网站内容读取

✅ **推荐做法**：
- 从官网首页开始，了解整体结构
- 优先阅读"Getting Started"和"Core Concepts"
- 记录所有重要链接
- 使用 WebFetch 批量获取相关页面

❌ **避免做法**：
- 只阅读单个页面就下结论
- 忽略 API 参考文档
- 不记录来源链接
- 跳过示例代码

### 技术特点分析

✅ **推荐做法**：
- 从多个维度分析技术特点
- 对比同类技术
- 关注实际应用场景
- 记录优缺点

❌ **避免做法**：
- 只看官方宣传
- 忽略社区反馈
- 不考虑学习曲线
- 不评估生态系统

### 知识规划

✅ **推荐做法**：
- 明确划分优先级
- 为每个主题提供多个链接
- 估算学习时间
- 保持规划灵活性

❌ **避免做法**：
- 优先级不明确
- 链接单一或缺失
- 不切实际的时间估算
- 规划过于僵化

### 迭代学习

✅ **推荐做法**：
- 至少执行 2 轮迭代
- 使用多种搜索查询
- 优先官方资源
- 记录每轮迭代结果

❌ **避免做法**：
- 一轮迭代就结束
- 只搜索一次
- 依赖非官方资源
- 不记录迭代过程

### 技能规划

✅ **推荐做法**：
- 技能职责单一
- 依赖关系清晰
- 难度递进合理
- 覆盖核心知识点

❌ **避免做法**：
- 技能职责混杂
- 依赖关系混乱
- 难度跳跃大
- 遗漏重要知识点

## 常见问题

### Q1: 如何判断信息完整性是否足够？

**答**：使用完整性评估公式，达到 80% 以上即可开始技能规划。核心标志：
- 所有核心概念已了解
- 主要 API 已掌握
- 有实际案例支撑
- 了解最佳实践

### Q2: 迭代学习应该执行几轮？

**答**：通常 2-3 轮即可：
- 第 1 轮：核心概念（60%）
- 第 2 轮：重要功能（80%）
- 第 3 轮：高级特性（90%+）

### Q3: 如何处理技术文档过大的情况？

**答**：采用分而治之策略：
1. 先阅读概览和快速开始
2. 识别核心章节
3. 优先学习核心内容
4. 其他内容作为后续迭代补充

### Q4: 如何确定技能拆分的粒度？

**答**：遵循以下原则：
- 每个技能学习时间 2-4 小时
- 每个技能对应一个明确模块
- 技能间依赖清晰
- 便于并行生成

### Q5: 搜索时使用什么语言？

**答**：根据技术特点选择：
- 国际主流技术：使用英文搜索
- 国内特有技术：使用中文搜索
- 不确定时：中英文都搜索

## 总结

智能网站分析模式通过系统化的流程，帮助快速从技术网站提取知识并生成技能拆分计划：

1. **网站内容读取**：使用 WebFetch/Read 获取内容
2. **技术特点分析**：多维度分析技术特性
3. **知识规划**：生成带链接的学习清单
4. **迭代学习**：通过 search 补充完善
5. **技能规划**：生成可执行的技能拆分计划

遵循本指南，可以高效地完成技术分析和技能生成任务。

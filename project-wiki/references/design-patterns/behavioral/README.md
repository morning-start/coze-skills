# 行为型模式 (Behavioral Patterns)

行为型模式负责对象间的高效沟通和职责委派。

## 模式列表

### 1. 责任链模式 (Chain of Responsibility)
- **定义**：使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合
- **关键点**：请求沿链传递，直到被处理
- **适用场景**：多个对象可以处理同一请求，但具体处理者由运行时决定
- [查看详细文档](./chain-of-responsibility.md)

### 2. 命令模式 (Command)
- **定义**：将请求封装为一个对象，从而可以用不同的请求对客户进行参数化
- **关键点**：分离调用者和接收者，支持撤销
- **适用场景**：需要将请求排队、记录日志或支持撤销操作
- [查看详细文档](./command.md)

### 3. 解释器模式 (Interpreter)
- **定义**：给定一个语言，定义它的文法表示，并定义一个解释器
- **关键点**：解释语言中的句子
- **适用场景**：需要解释特定领域语言
- [查看详细文档](./interpreter.md)

### 4. 迭代器模式 (Iterator)
- **定义**：提供一种方法顺序访问聚合对象中的各个元素，而无需暴露其内部表示
- **关键点**：统一遍历接口
- **适用场景**：需要遍历聚合对象，但不希望暴露其内部结构
- [查看详细文档](./iterator.md)

### 5. 中介者模式 (Mediator)
- **定义**：用一个中介对象来封装一系列的对象交互
- **关键点**：集中管理通信，多对多变一对一
- **适用场景**：对象间通信复杂，产生混乱的依赖关系
- [查看详细文档](./mediator.md)

### 6. 备忘录模式 (Memento)
- **定义**：在不破坏封装性的前提下，捕获一个对象的内部状态
- **关键点**：保存和恢复对象状态
- **适用场景**：需要提供撤销功能，或需要保存对象状态
- [查看详细文档](./memento.md)

### 7. 观察者模式 (Observer)
- **定义**：定义对象间的一种一对多的依赖关系
- **关键点**：自动通知更新，发布-订阅机制
- **适用场景**：一个对象的状态变化需要通知其他对象
- [查看详细文档](./observer.md)

### 8. 状态模式 (State)
- **定义**：允许对象在其内部状态改变时改变它的行为
- **关键点**：状态决定行为，消除条件分支
- **适用场景**：对象的行为依赖于它的状态
- [查看详细文档](./state.md)

### 9. 策略模式 (Strategy)
- **定义**：定义一系列的算法，把它们一个个封装起来，并且使它们可相互替换
- **关键点**：算法族，可互换
- **适用场景**：需要在不同算法间切换
- [查看详细文档](./strategy.md)

### 10. 模板方法模式 (Template Method)
- **定义**：定义一个操作中的算法的骨架，而将一些步骤延迟到子类中
- **关键点**：算法骨架，延迟实现
- **适用场景**：算法的整体结构固定，但某些步骤需要变化
- [查看详细文档](./template-method.md)

### 11. 访问者模式 (Visitor)
- **定义**：表示一个作用于某对象结构中的各元素的操作
- **关键点**：分离操作和对象结构
- **适用场景**：需要对对象结构中的元素进行多种不同操作
- [查看详细文档](./visitor.md)

## 行为型模式分类

### 类行为型模式（使用继承）
- 模板方法模式
- 解释器模式

### 对象行为型模式（使用组合）
- 责任链模式
- 命令模式
- 迭代器模式
- 中介者模式
- 备忘录模式
- 观察者模式
- 状态模式
- 策略模式
- 访问者模式

## 核心设计原则

- **单一职责原则**：一个类只负责一项职责
- **开闭原则**：对扩展开放，对修改关闭
- **依赖倒转原则**：依赖抽象，不依赖具体

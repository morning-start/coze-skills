# 创建型模式 (Creational Patterns)

创建型模式提供了创建对象的机制，能够提升已有代码的灵活性和可复用性。

## 模式列表

### 1. 单例模式 (Singleton)
- **定义**：确保一个类只有一个实例，并提供一个全局访问点
- **关键点**：只有一个实例、自我实例化、全局访问点
- **适用场景**：系统中只需要一个实例，如配置管理器、连接池
- [查看详细文档](./singleton.md)

### 2. 工厂方法模式 (Factory Method)
- **定义**：定义创建对象的接口，让子类决定实例化哪个类
- **关键点**：将实例化推迟到子类
- **适用场景**：需要灵活创建不同类型对象时
- [查看详细文档](./factory-method.md)

### 3. 抽象工厂模式 (Abstract Factory)
- **定义**：创建相关或依赖对象的家族，无需明确指定具体类
- **关键点**：创建产品族，保持产品一致性
- **适用场景**：需要创建一系列相关产品时
- [查看详细文档](./abstract-factory.md)

### 4. 建造者模式 (Builder)
- **定义**：将复杂对象的构建与表示分离
- **关键点**：按步骤构造，分离构建过程
- **适用场景**：对象结构复杂，构建过程需要灵活控制
- [查看详细文档](./builder.md)

### 5. 原型模式 (Prototype)
- **定义**：通过复制现有实例创建新实例
- **关键点**：克隆对象，避免复杂构造
- **适用场景**：创建对象成本大，需要频繁创建相似对象
- [查看详细文档](./prototype.md)

## 创建型模式对比

| 模式 | 主要目的 | 复杂度 | 灵活性 |
|------|---------|--------|--------|
| 单例模式 | 确保唯一实例 | ⭐ | ⭐⭐ |
| 工厂方法 | 延迟实例化 | ⭐⭐ | ⭐⭐⭐ |
| 抽象工厂 | 创建产品族 | ⭐⭐⭐ | ⭐⭐ |
| 建造者模式 | 分步构建复杂对象 | ⭐⭐⭐ | ⭐⭐⭐ |
| 原型模式 | 克隆现有对象 | ⭐⭐ | ⭐⭐⭐ |

## 核心设计原则

- **对扩展开放，对修改关闭**（开闭原则）
- **依赖抽象，不依赖具体**（依赖倒转原则）
- **优先使用对象组合，而非继承**（合成复用原则）

# 文档生命周期管理指南

## 概述

文档生命周期管理确保文档从创建到归档的全过程可追踪、可控制，保证文档质量。

## 生命周期状态

| 状态 | 符号 | 说明 |
|------|------|------|
| Draft | 🟡 | 草稿阶段，内容正在编写 |
| Review | 🟠 | 审核阶段，等待评审 |
| Published | 🟢 | 已发布，当前版本可用 |
| Maintenance | 🔵 | 维护中，持续更新 |
| Archived | 🟣 | 已归档，不再维护 |
| Deprecated | 🔴 | 已废弃，不推荐使用 |

## 状态详细说明

### Draft（草稿）
- **进入条件**：新文档创建
- **操作**：
  - 编写文档内容
  - 添加基本结构
  - 标注待完善项
- **退出条件**：内容基本完成，提交审核

### Review（审核）
- **进入条件**：作者提交审核
- **操作**：
  - 技术准确性审核
  - 格式规范性审核
  - 完整性审核
- **退出条件**：审核通过/需修改/拒绝

### Published（已发布）
- **进入条件**：审核通过
- **操作**：
  - 文档可供查阅
  - 链接可用
- **退出条件**：需要更新/归档

### Maintenance（维护中）
- **进入条件**：文档需要更新
- **操作**：
  - 修复错误
  - 补充内容
  - 更新过时信息
- **退出条件**：更新完成/归档/废弃

### Archived（已归档）
- **进入条件**：文档不再维护
- **操作**：
  - 保留历史版本
  - 标记为归档
  - 移除导航链接
- **退出条件**：废弃

### Deprecated（已废弃）
- **进入条件**：功能或项目废弃
- **操作**：
  - 标记废弃说明
  - 提供迁移指南
  - 设置重定向（如需要）

## 执行流程

### 1. 创建文档

```
1. 确定文档类型和位置
2. 选择合适的模板
3. 填写基本信息（作者、创建日期、版本）
4. 设置状态为 Draft
```

### 2. 状态转换

每个状态转换需要记录：
- 转换时间
- 操作人
- 转换原因
- 备注（如有）

### 3. 版本管理

- 每次状态转换或重大修改需要更新版本号
- 保留历史版本记录
- 使用 CHANGELOG 记录变更

## 文档元信息

每个文档应包含元信息头：

```markdown
---
title: 文档标题
author: 作者名
created: 2024-01-01
updated: 2024-01-15
version: 1.0.0
status: Published  # Draft/Review/Published/Maintenance/Archived/Deprecated
reviewers: ["审阅者1", "审阅者2"]
---
```

## 注意事项

- 文档必须经过 Review 才能发布
- 定期检查 Maintenance 状态文档的时效性
- Archived 和 Deprecated 文档仍需保留
- 状态变更需要通知相关人员

# 知识搜索指南

## 概述

知识搜索是 ProjectWiki 的核心能力之一，用于帮助 Agent 快速获取不熟悉技术栈的知识，并积累到项目知识库中。

## 搜索场景

- **技术选型**：了解新技术的优缺点、适用场景
- **问题排查**：搜索特定技术的常见问题和解决方案
- **学习研究**：快速掌握某个技术栈的核心概念和最佳实践
- **文档编写**：获取准确的技术术语和实现细节

## 搜索类型

| 类型 | 说明 | 典型关键词 |
|------|------|-------------|
| `library` | 库/框架使用 | 官方文档、API、使用示例 |
| `architecture` | 架构设计 | 架构模式、设计原理 |
| `pattern` | 设计模式 | 实现方式、代码示例 |
| `principle` | 原理概念 | 核心概念、工作原理 |
| `math` | 数学原理 | 公式推导、应用场景 |

## 执行流程

### 1. 识别搜索需求

当遇到以下情况时，需要进行知识搜索：
- 用户提到不熟悉的技术栈
- 项目需要引入新的依赖
- 遇到技术难点需要参考方案

### 2. 执行 Web 搜索

使用 `web_search` 工具搜索相关知识：

```
搜索关键词 = 技术栈名称 + 知识类型关键词

示例：
- "React 官方文档 教程 最佳实践"
- "Docker 架构设计 模式"
- "Python asyncio 原理 核心概念"
```

### 3. 整理知识内容

将搜索结果整理为结构化文档，保存到 `wiki/references/` 目录：

```
wiki/references/
├── tech/                    # 技术知识
│   ├── react/
│   │   └── overview.md
│   └── docker/
│       └── architecture.md
├── patterns/               # 设计模式
└── principles/            # 原理概念
```

### 4. 更新知识索引

在 `references/knowledge/` 下维护知识索引，便于后续查询。

## 知识文档结构

每个知识文档应包含：

```markdown
# 技术名称

## 概述
一句话介绍

## 核心概念
- 概念1: 说明
- 概念2: 说明

## 使用场景
- 场景1
- 场景2

## 最佳实践
1. 实践1
2. 实践2

## 注意事项
- 注意1
- 注意2

## 参考资源
- [官方文档](url)
- [教程](url)
```

## 注意事项

- 搜索结果必须保存到 `wiki/references/` 目录
- 优先选择权威来源（官方文档、知名博客）
- 注明知识来源和获取日期
- 定期更新已有知识，防止过时

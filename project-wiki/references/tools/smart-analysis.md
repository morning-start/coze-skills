# 智能分析

智能分析是 ProjectWiki 的核心能力之一，通过自动分析项目来提供准确的文档推荐和优化建议。

## 功能概述

| 功能 | 说明 |
|------|------|
| 项目类型检测 | 自动识别项目使用的框架和技术栈 |
| 复杂度评估 | 评估项目复杂度（简单/中等/复杂） |
| 文档推荐 | 根据项目类型推荐需要生成的文档 |
| 角色推荐 | 根据文档类型推荐相关角色视图 |
| 知识缺口 | 识别项目中缺失的知识 |
| 优化建议 | 生成项目结构优化建议 |

## 支持的框架

### 后端框架
- Django、Flask、FastAPI、Spring Boot、Gin

### 前端框架
- React、Vue、Svelte、SolidJS

### 跨平台框架
- Flutter、Electron、Tauri、Wails

## 执行流程

### 1. 检测项目框架

扫描项目文件，识别技术栈：

```
检测依据：
- Django: manage.py, settings.py, urls.py
- Flask: app.py, requirements.txt
- FastAPI: main.py, dependencies.py
- React: package.json, src/App.js
- Vue: package.json, src/main.js
- Flutter: pubspec.yaml, lib/main.dart
```

### 2. 评估项目复杂度

根据以下因素评估复杂度：
- 代码文件数量
- 目录层级深度
- 模块数量
- 依赖复杂度

### 3. 推荐文档类型

根据框架类型推荐文档：

| 框架 | 推荐文档 |
|------|----------|
| Django | API、模块、架构、数据流 |
| Flask | API、模块、服务 |
| FastAPI | API、模块、数据流 |
| React | 功能、模块、架构 |
| Vue | 功能、模块、架构 |
| Flutter | 功能、模块、架构 |

### 4. 生成优化建议

根据分析结果生成：
- 目录结构优化建议
- 缺失文件提醒
- 模块划分建议

## 使用场景

### 场景 1：新项目初始化

```
1. 分析项目结构
2. 确定项目类型
3. 获取文档推荐
4. 生成基础文档
```

### 场景 2：项目评估

```
1. 评估项目复杂度
2. 识别知识缺口
3. 生成评估报告
```

### 场景 3：文档补全

```
1. 分析现有文档
2. 识别缺失内容
3. 推荐补充文档

## 注意事项

- 分析结果仅作参考，实际情况需人工确认
- 框架检测基于文件特征，可能存在误判
- 建议定期更新分析以反映项目最新状态
